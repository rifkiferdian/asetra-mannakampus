Approval Steps

Tentukan urutan approver berdasarkan role dan scope.

Panduan memilih scope approval step

STORE: sistem mencari approver dari menu Store Approver berdasarkan store PR. Contoh: PR dari MK4 Palagan + role manager akan mencari manager yang dimapping untuk MK4 Palagan.

HO: sistem mencari user aktif yang punya role tersebut di User Management. Tidak perlu data Store Approver. Cocok untuk finance-manager, GM, atau approver pusat.

Contoh umum: Store Manager pakai STORE, Finance Manager pakai HO, GM pusat pakai HO.

Parallel = Yes: beberapa approver pada step order yang sama bisa berjalan bersamaan.

Required = Yes: approver ini wajib approve agar proses bisa lanjut.

Catatan: jika scope salah, workflow bisa gagal dibuat karena sistem mencari approver dari sumber yang berbeda.
